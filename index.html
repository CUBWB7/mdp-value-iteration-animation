2. 本题的特殊化：把 R 从求和里提出来

本题 reward 的定义是「站到哪格就领哪格的奖励」（普通 −2 / 红 −1 / 绿 +1）： 注意细节：作业用的是「进入这格后立得 R(s)」的写法，相当于把 R 挂在当前 state s 上，与动作 a、下一格 s' 都无关。

3. 用作「赋值」（迭代式）：把 V* 拆成 V_old 和 V_new

方程左右两边的 V* 是同一个函数，求解时用「不动点迭代」：右边代入旧值 V_old， 算出来的结果赋值给左边的 V_new，再把 V_new 当作下一轮的 V_old。这就是 Value Iteration 的核心：

4. 引入 Q(s, a)：把「某个动作下的期望未来价值」单独命名

右边那个 Σ T(s,a,s')·V_old(s') 是「在动作 a 下，下一步价值的期望」， 为了书写简洁，给它一个名字 Q(s, a)：

于是公式简化成「先对 4 个动作各算一个 Q，再选最大那个」：

下方信息栏里的 Q(UP) / Q(DOWN) / Q(LEFT) / Q(RIGHT) 就是这 4 个 Q 值； 其中最大的那个对应的动作就是 a*（公式里的 max_a）。

直觉版：「这格的最优价值」= 「站这格立刻拿的奖励」+ 折扣 × 「在最聪明的动作下，未来的期望价值」。 标准 Value Iteration 把这个等式当赋值反复用直到 V 不再变； 本题特殊：不迭代到收敛，只沿 trajectory 顺序对每个 state 做一次 backup。
