# 题目解法 · MDP Value Iteration on Grid World

> 配套 PDF：`week10_mdp_rl_tutorial_exercise.pdf`（题目）/ `week10_mdp_rl_tutorial_solution.pdf`（答案）

## 一、题目设定（用大白话讲）

想象一个 4 列 3 行的房间，机器人从左下角出发。房间里有：

```
┌────┬────┬────┬────┐
│    │    │    │ +1 │   ← 第 2 行（绿色：得 +1 分，到这里游戏结束）
├────┼────┼────┼────┤
│    │墙 │    │ -1 │   ← 第 1 行（红色：得 -1 分；黑色：墙，进不去）
├────┼────┼────┼────┤
│ 起 │    │    │    │   ← 第 0 行（机器人从 [0,0] 出发）
└────┴────┴────┴────┘
  col0 col1 col2 col3
```

**坐标记法**：`[行, 列]`，左下角 `[0,0]`，右上角 `[2,3]`。注意行号自下而上递增。

**奖励**：
- 进绿色（[2,3]）：+1，游戏结束
- 进红色（[1,3]）：−1
- 进其它（普通格）：−2
- 墙（[1,1]）：进不去

**移动规则**（这里有随机性）：

机器人每步只能选 4 个动作之一：上 / 下 / 左 / 右。但它**不会精确执行**：

- 80% 概率往你想去的方向走
- 各 10% 概率向**垂直**方向偏（比如想往右走，就有 10% 偏到上、10% 偏到下）
- 如果偏移会撞墙或撞边界，那机器人**留在原地**

**折扣因子** γ = 0.8（未来的奖励要打折，越远越不值钱）

**目标**：用 Bellman 方程算出每个格子的「价值」V，即「从这格出发预期能赚多少分」。

---

## 二、核心公式

**Bellman 最优更新公式**：

$$
V_\text{new}(s) = R(s) + \gamma \cdot \max_a \sum_{s'} T(s, a, s') \cdot V_\text{old}(s')
$$

通俗念法：

> **「这格新的价值 V」= 「这格本身的奖励 R」+ 0.8 × 「在 4 个动作中挑一个最好的，那个动作下、3 种可能去向各自的旧 V 加权平均」**

里面每一项的意思：

- `R(s)`：站在这格立刻拿到的奖励（普通 -2，绿 +1，红 -1）
- `γ = 0.8`：折扣因子（未来打折）
- `max_a`：在 4 个动作里挑 Q 值最大的那个动作 `a*`
- `T(s, a, s')`：转移概率（0.8 / 0.1 / 0.1 中的一个）
- `V_old(s')`：转移到的下一格的旧 V 值（没访问过的格子默认 0）

---

## 三、本题的算法（不是收敛式 value iteration！）

很多人看到 value iteration 会以为「全图反复刷直到收敛」。**本题不是这样**：

> 老师给定 6 条 trajectory（具体步骤序列），要求**沿 trajectory 顺序**，每访问一个 state 就用上面的公式做 **one-step backup**，每个 state 每访问一次只更新一次，不收敛、不再回过头。

并且：

- **V 表跨 trajectory 累积**：Trajectory 2 开始时，V 表用的是 Trajectory 1 跑完后的状态，不重置
- 一个 state 在不同 trajectory 里被多次访问 → 每次都按当时的 V_old 更新一次

### ⚠️ 最容易踩的坑：trajectory 表里的 `a_t` ≠ 公式里的 `a`

题目给的 trajectory 表每步都写明了 `a_t`（比如 Trajectory 2 step 0 写 `a_0 = UP`）。**但这个 `a_t` 不是用来代入 Bellman 公式的！**

公式里的 `a` 是 `max_a`，意思是「无视 trajectory 写了什么动作，自己在 4 个动作里挑 Q 最大的」。

`a_t` 唯一的作用是：**告诉动画「机器人这一步走到哪一格」**（即决定 `s_{t+1}`）。

#### 验证这个坑

T2 step 0 在 `s = [0,0]`，T1 跑完后 V 表是：

| 格子 | V_old |
|---|---|
| [0,0], [1,0], [2,0], [2,1], [2,2] | −2 |
| [2,3] | +1 |
| 其它（[0,1], [0,2], [0,3], [1,2], [1,3]） | 0 |

算 4 个 Q：

| 动作 a | 主邻居 (0.8) | 偏 1 (0.1) | 偏 2 (0.1) | Q(a) |
|---|---|---|---|---|
| UP    | [1,0]=−2  | 出界停留=−2 | [0,1]=0 | 0.8·−2 + 0.1·−2 + 0.1·0 = **−1.8** |
| DOWN  | 出界停留=−2 | 出界停留=−2 | [0,1]=0 | 0.8·−2 + 0.1·−2 + 0.1·0 = **−1.8** |
| LEFT  | 出界停留=−2 | 出界停留=−2 | [1,0]=−2 | 0.8·−2 + 0.1·−2 + 0.1·−2 = **−2.0** |
| RIGHT | [0,1]=0   | [1,0]=−2   | 出界停留=−2 | 0.8·0 + 0.1·−2 + 0.1·−2 = **−0.4** |

`max_a Q = −0.4`（动作 RIGHT），所以：

$$V_\text{new}([0,0]) = -2 + 0.8 \cdot (-0.4) = -2.32$$

✓ 和答案 PDF 一致。**注意 trajectory 表写的是 `a_0 = UP`，但公式里挑的是 RIGHT**。这就是 `a_t ≠ a*` 的具体例子。

### 每一步拆成 6 个 Sub-step（普通格）

把上面这套流程拆开看，每访问一格其实是按固定顺序做 6 件事——这也是动画里每个 step 的 6 帧：

| 子步 | 名称 | 做什么 |
|---|---|---|
| 0 | **Idle** | 机器人停在当前格 s，淡灰提示 trajectory 写的 `a_t`（仅用于走位，不进公式） |
| 1 | **Q-table** | 对 4 个动作各算 Q(a) = 0.8·V(主) + 0.1·V(偏1) + 0.1·V(偏2)；撞墙/出界用自己的 V_old |
| 2 | **Pick max** | 4 个 Q 里挑最大那个，记为 a\*（可能 ≠ a_t） |
| 3 | **Highlight** | grid 上高亮 a\* 对应的 3 个 stochastic 邻居（深色 0.8，浅色 0.1×2） |
| 4 | **Substitute** | 把数字代进公式：`V_new = R(s) + 0.8 · Q(a*)` |
| 5 | **Commit & Move** | V_new 写进 V 表那一格；机器人按 **a_t**（不是 a\*）走到下一格 |

**Terminal 格（绿 [2,3] / 红 [1,3]）只有 2 个 sub-step**：进入即结束，直接 V = R（绿 +1 / 红 −1），无 Q 计算、无后续移动。T3/T4 因为第 5 步进红格而提前结束（只 5 步，其它 6 步）。

## 四、Trajectory 1 的逐步推算（作业入门）

T1 的 6 步：[0,0] → [1,0] → [2,0] → [2,1] → [2,2] → [2,3]，全部动作正好被选中执行。

T1 是最简单的一条，因为开始时**所有格子的 V_old 都是 0**。所以每步算 V：

$$V_\text{new}(s) = R(s) + \gamma \cdot \max_a (\underbrace{0.8 \cdot 0 + 0.1 \cdot 0 + 0.1 \cdot 0}_{= 0}) = R(s)$$

| Step | s | V_new |
|---|---|---|
| 0 | [0,0] | −2 |
| 1 | [1,0] | −2 |
| 2 | [2,0] | −2 |
| 3 | [2,1] | −2 |
| 4 | [2,2] | −2 |
| 5 | [2,3] | +1（绿色 / Terminal，进入即结束）|

T1 跑完后，V 表里这 6 个格子有值，剩下 5 个普通格 + 红色仍是 0（默认）。

## 六、Bellman 直觉小记

为什么 V([0,0]) 越算越小（−2 → −2.32 → ...）？

因为**每访问一次都要被本格 reward (−2) 拖累一次**。在 trajectory 反复回到同一格的过程中，−2 的 reward 像「过路费」一样反复累计进 V，最终 V 反映的是「**预期还要被罚多少分才能走到 +1**」。

这种沿 trajectory 的累积更新，本质上是**异步 / Gauss-Seidel 风格**的 value iteration——比同步更新更接近真实学习过程（agent 实际走过哪里就更新哪里）。

## 八、能不能靠直觉省掉一些计算？

每步都老老实实算 4 个 Q 太慢，**确实可以靠直觉粗筛**，但要避开几个陷阱。

### 主方向法则（最有用的捷径）

$$Q(s, a) = \underbrace{0.8 \cdot V(\text{主方向})}_{权重最大} + 0.1 \cdot V(\text{偏 1}) + 0.1 \cdot V(\text{偏 2})$$

80% 的权重压在主方向，所以**先看每个动作主方向那格的 V_old，挑最大那个**，往往就是 a*。
T1 全程都符合这条：哪个方向更靠近绿格，那个就是 a*。

### 直觉会翻车的 3 类陷阱

**1. 主方向 V 接近时，0.1 偏移格说了算**

T2 step 0 在 [0,0]：T1 跑完后 UP 的主方向 [1,0] 与 RIGHT 的主方向 [0,1] **V_old 都接近 −2**，看似打平。但 RIGHT 的偏移格里一个是出界停留 V=−2，一个是 [1,0]=−2；UP 的偏移格里有个 [0,1]=0 —— 算下来反而 RIGHT 的 Q 略高。

**2. 靠墙时「撞墙 = 留在原地」会用自己的 V_old 当邻居**

边界格的某个 0.1 偏移变成「停在自己」，等于 0.1·V(s)。当前格 V_old 越糟，这个动作越被拖累；反过来当前格还行就没事。

**3. 红格附近，0.1 侧滑进 V=−1 会拉低 Q**

主方向看起来不错的动作，可能因为 0.1 偏移正好滑进红格，被另一个看似差不多的动作反超。

### 实战策略

1. **粗筛**：4 个动作只看主方向 V_old，淘汰明显差的 2 个
2. **精算**：剩下 1–2 个候选才完整算 Q
3. **打平警告**：候选主方向 V_old 相差小于 0.5 时，老实算全式，别凭感觉
4. **永远记住**：a* ≠ a_t，挑出 a* 算 V_new，但机器人下一步走的方向是 trajectory 写的 a_t
